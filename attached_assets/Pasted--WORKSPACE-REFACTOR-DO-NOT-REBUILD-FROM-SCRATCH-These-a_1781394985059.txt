# WORKSPACE REFACTOR - DO NOT REBUILD FROM SCRATCH

These are targeted changes to the existing platform.

Do NOT redesign the entire system.

Apply only the modifications below.

---

# REMOVE AI AGENT TAB

Current:

Chat
Files
Preview
Deploy
Logs
History

AI Agent

Problem:

AI Agent behaves like a normal chatbot.

It outputs raw code.

It does not create files.

It does not update workspace.

It does not update preview.

It does not update deployments.

It is disconnected from the project.

---

Required:

Remove AI Agent tab completely.

Replace it with the existing Build experience.

The Build interface becomes the primary AI assistant.

---

# BUILD BECOMES THE AI AGENT

Current Build:

User enters prompt.

AI generates project.

Build finishes.

Interaction ends.

---

Required:

Build should become a persistent project assistant.

Example:

User:

Build a restaurant ordering platform.

AI builds project.

Later:

Add dark mode.

AI modifies project files.

Later:

Add payments.

AI modifies project files.

Later:

Deploy to my server.

AI performs deployment.

The Build panel must continue conversations.

Build is now the project AI.

No separate AI Agent needed.

---

# AI MUST WRITE TO FILES

Current:

AI writes code inside chat.

Example:

index.html
style.css
app.js

inside messages.

---

Required:

AI NEVER outputs full files inside chat.

Instead:

AI creates files directly.

Files appear immediately inside:

Files Tab

Explorer

Project Storage

Preview

Deployment Package

Chat only explains what changed.

Example:

Created:

* index.html
* style.css
* app.js

Preview updated.

Deployment ready.

---

# FILE MODIFICATION MODE

Current:

User says:

Change button color.

AI rewrites entire project.

Consumes tokens.

Expensive.

Slow.

Breaks code.

---

Required:

AI must perform targeted edits.

Example:

User:

Change login button color to blue.

AI:

Modified:
frontend/components/LoginButton.tsx

Only.

Do not regenerate unaffected files.

Do not rebuild project.

Do not recreate files.

---

# DIFF-BASED EDITING

Before changing files:

AI loads current file.

Calculates difference.

Applies patch.

Saves file.

Only changed content is rewritten.

This dramatically reduces:

* token usage
* build time
* costs
