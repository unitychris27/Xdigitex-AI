## Root Cause

Your agent is acting like:

> "I will solve everything at once."

Instead it should act like:

> "One task → verify → next task."

OpenHands, Manus, Devin and Replit Agent all work because they break work into small verified loops.

Your agent is currently:

1. Reading files
2. Reading configs
3. Reading middleware
4. Testing MySQL
5. Creating users
6. Modifying code
7. Building UI

all in one execution.

That destroys context.

# Fix #1: Hard Limit Agent Planning

Add system rule:

```text
NEVER create plans longer than 5 steps.

If task exceeds 5 steps:

STOP
COMPLETE CURRENT STEP
VERIFY
CONTINUE

Do not attempt entire project in one execution.
```

Bad:

```text
Build advertising network.
Create DB.
Create UI.
Create API.
Create Auth.
Create Admin.
Create Publisher.
Create Payments.
```

Good:

```text
Current Goal:
Fix login issue

Step 1:
Read Auth.php

Step 2:
Read Database.php

Step 3:
Find error

Step 4:
Patch error

Step 5:
Verify login
```

Then stop.

# Fix #4: Prevent Massive File Reads

I see:

```bash
cat Database.php
cat Auth.php
```

Then later:

```bash
cat entire project
```

This kills context.

Rule:

```text
Never read entire files over 500 lines.

Use:

head
tail
grep
sed

Read only relevant sections.
```

Example:

```bash
grep login Auth.php
```

instead of:

```bash
cat Auth.php
```

```text
You are an autonomous software engineer.

Rules:

- Complete only ONE objective at a time.
- Maximum 5 planning steps.
- Maximum 15 commands per execution.
```
