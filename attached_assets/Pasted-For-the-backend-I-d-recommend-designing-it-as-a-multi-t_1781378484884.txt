For the backend, I'd recommend designing it as a **multi-tenant AI orchestration platform** rather than hard-coding OpenAI, Anthropic, DeepSeek, etc. The admin should be able to add, remove, rotate, and route models without redeploying the application.

# Backend System Prompt

Build a production-grade backend for an AI Multi-Agent Development Platform.

The platform allows users to:

* Create AI coding projects
* Run multiple AI agents simultaneously
* Deploy Telegram bots
* Manage secrets
* Connect SSH servers
* Deploy applications
* Track billing and usage
* Earn referral commissions
* Use marketplace templates
* Manage teams and permissions

The backend must support:

* Multi-tenancy
* RBAC permissions
* Real-time events
* AI model orchestration
* Agent execution
* Billing
* Auditing
* API key management

One architectural improvement I'd make: treat **AI providers as plugins**. Then when your admin wants to add a new provider such as [OpenAI](https://platform.openai.com?utm_source=chatgpt.com), [Anthropic](https://www.anthropic.com?utm_source=chatgpt.com), [DeepSeek](https://www.deepseek.com?utm_source=chatgpt.com), [Google AI Studio](https://aistudio.google.com?utm_source=chatgpt.com), or future providers, they simply add credentials and model definitions through the admin panel. The agent system continues to work without backend code changes because it talks only to the AI Orchestrator layer. This will save a huge amount of maintenance as the model ecosystem evolves.
