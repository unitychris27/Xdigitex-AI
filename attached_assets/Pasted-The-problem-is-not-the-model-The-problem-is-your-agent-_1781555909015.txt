The problem is not the model. The problem is your **agent execution strategy**.

Your prompt is asking the AI to generate:

* 20+ database tables
* 100+ PHP files
* Authentication
* Advertiser dashboard
* Publisher dashboard
* Ad server
* Wallets
* Payments
* AI modules
* API docs
* Deployment guide

in **one task**.

No coding agent (OpenHands, OpenDevin, Claude Code, Cursor, Roo Code, Cline, Aider) will reliably finish that in one run.

What happens:

```text
User Prompt
      ↓
Planner creates huge plan
      ↓
Agent starts writing
      ↓
Context grows
      ↓
Token limit reached
      ↓
Agent loses state
      ↓
Stops halfway
```

---

# Fix 1: Force Phase-Based Development

Never allow:

```text
Build complete advertising network
```

Instead:

```text
PHASE 1

Create:

- Database schema
- MVC framework
- Authentication
- Admin login

Nothing else.

When complete:
- Test
- Fix errors
- Commit

Stop.
```

Then:

```text
PHASE 2

Create:

- Advertiser dashboard
- Campaign CRUD
- Ad CRUD

Nothing else.

Stop.
```

Then:

```text
PHASE 3

Create:

- Publisher dashboard
- Websites
- Ad zones

Stop.
```

Then:

```text
PHASE 4

Create:

- Tracking
- Impressions
- Clicks
- Conversions

Stop.
```

This alone increases success rate dramatically.

# Fix 2: Hard File Limits

Add this to your system prompt:

```text
Never generate more than 20 files in a single execution.

If more files are needed:

1. Create first batch
2. Verify batch
3. Save state
4. Ask system for next batch

Continue until complete.
```

```text
Load project_tasks
```

instead of starting over.

```json
  ],
  "completed_tasks":[
    "Authentication"
  ],
  "next_task":"Admin Dashboard"

```

When AI restarts:

```text

Continue
```

# Fix 5: Force Patch Mode

Currently your AI does:

```text
Rewrite whole file
```

This is terrible.

Instead:

```text
User:
Add forgot password.

Agent:

Files affected:
- AuthController.php
- login.php

Patch only these files.

Do not rewrite unrelated code.
```

Use an Aider-style diff approach:

```diff
+ public function forgotPassword()
{
}
```

instead of replacing entire files.



### The rule I'd add immediately

```text
If estimated project size exceeds 50 files:

DO NOT BUILD ENTIRE PROJECT.

Create execution plan.

Build only next phase.

Verify.

Save checkpoint.

Stop.

Wait for continuation.
```

That single rule usually fixes 70–80% of "agent started coding then stopped" problems.
